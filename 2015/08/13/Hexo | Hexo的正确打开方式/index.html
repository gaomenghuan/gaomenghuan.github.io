Hexo | Hexo的正确打开方式

  • Hexo

如果你和我一样是技术小白…

I assume你和我一样，许多基本起步工具都还没有配置好。下面先一个一个来安装：

  • Xcode

Xcode是苹果的IDE开发环境。安装了它，你的Mac才能编译东西。

  • Homebrew

Homebrew则是Mac OS X的软件包管理工具。它可以用命令行来控制安装、升级和移除软件。

  • Git

Git，用Linus Torvalds的话来解释就懂了——“It’s the stupid content tracker.” 说简单点，就是程序员们发布和交流源代码的地方。

  • Node.js

Node是一个js的运行环境。为什么要装这个呢？因为Hexo就是基于Node.js开发哒。

具体安装方法就不说了，百度一搜一大堆。

安装Hexo

Hexo的正常安装方法可在Hexo Documentation里查阅。下面只说说我遇到的一个坑：

npm安装Hexo报错

简直不晓得该说什么了，因为这个，重装了无数次Node，担心是Node没安装完整的原因。然后用npm测试安装别的比如express，都是可以的啊啊啊！最后解决方式是：

使用

sudo npm install --unsafe-perm --verbose -g hexo-cli

替换原始命令

npm install hexo-cli -g

使用Hexo

  1. 先到GitHub注册一个账号，免费申请一个repository，用来存放你的Hexo博客内容。

  2. 然后根据GitHub Pages的说明来创建你的Github pages.

  3. 再根据Hexo Deployment的说明将你的Hexo博客部署到GitHub Pages.

至此，你的博客就已经可以在你的username.github.io上看到啦。

But，如果你还想使用独立域名…

好吧，这就涉及到独立域名解析绑定至GitHub的内容了。

  1. 遵照GitHub Help的提示，创建CNAME文件，解析域名；

  2. 这里也是个坑！！！ 本来上面一步就能搞定的，结果我更新文章内容时，每次deploy都会把第一步创建的CNAME弄丢！然后就404了T_T！然后我就想啊想啊，妈哒，最后突然醒悟，把CNAME存放到Hexo本地的随便哪个文件夹比如source，那之后每次update也会同时上传CNAME了啊，就不会丢了啊。好傻！
